# cli-development

Best practices for building CLI applications across languages, covering design principles, argument parsing, output formatting, error handling, cross-platform considerations, testing, and documentation.

## Files

| File | Purpose |
|------|---------|
| `knowledge.md` | Full CLI development guidelines and patterns |

## Topics

- CLI design principles (Unix philosophy, verb-noun pattern, subcommands vs flags)
- Argument parsing, flags, environment variables, and configuration precedence
- User interface (help text, progress indicators, color output, interactive prompts)
- Output formatting (human-readable, JSON, YAML, CSV) and exit codes
- Error handling with clear messages, suggestions, and debug mode
- Cross-platform path handling, line endings, and terminal capabilities
- Testing strategies (integration tests, output verification, exit code checks)
- Common CLI libraries for Python, Node.js, Go, and Rust
