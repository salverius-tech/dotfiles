# structured-analysis

Structured analytical frameworks for analyzing artifacts (prompts, systems, documents, code, architecture). 12 frameworks in 3 tiers with post-plan validation workflow.

## Files

| File | Purpose |
|------|---------|
| `knowledge.md` | Full structured analysis guidelines and framework templates |
| `idempotency-audit.template` | Template for auditing idempotency of scripts and migrations |

## Topics

- Framework tiers: Core (4), Auto-Invoke (3), On-Demand (5)
- Technique templates (meta-prompting, deep-analyze, reasoning-scaffold, etc.)
- Adversarial review and scope boundary analysis
- Post-plan validation workflow
- Technique combinations and anti-patterns
- Effectiveness indicators
