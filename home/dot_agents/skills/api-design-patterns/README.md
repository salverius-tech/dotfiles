Language-agnostic API design patterns covering REST and GraphQL, including resource naming, HTTP methods, status codes, versioning, pagination, filtering, authentication, error handling, and schema design.

- Versioning strategies and pagination patterns
- Authentication and authorization (API keys, JWT, OAuth2)

- Request validation and error response formats
- Rate limiting and caching strategies
- OpenAPI/Swagger and GraphQL documentation
- Common pitfalls and solutions
