# least-astonishment

Principle of Least Astonishment (POLA) - ensure code changes behave as users and developers expect through predictable, convention-following changes.

## Files

| File | Purpose |
|------|---------|
| `knowledge.md` | Full POLA guidelines and patterns |

## Topics

- The Surprise Check for evaluating changes
- Common POLA violations (scope creep, hidden side effects, inconsistent naming)
- POLA-compliant patterns (minimal diffs, matching vocabulary, backward compatibility)
- When and how to make larger changes that break expectations
- Integration with KISS, DRY, and consistency principles
