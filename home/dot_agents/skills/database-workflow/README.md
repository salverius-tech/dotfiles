# database-workflow

Language-agnostic database best practices covering migrations, schema design, ORM patterns, query optimization, and testing strategies.

## Files

| File | Description |
|------|-------------|
| `knowledge.md` | Full database workflow guidelines and patterns |

## Topics

- Migration strategies (versioning, up/down, idempotent, rollback)
- Schema design (normalization, denormalization, indexing, keys, constraints)
- ORM patterns (Active Record vs Data Mapper, N+1 prevention, eager loading)
- Query optimization (EXPLAIN analysis, index usage, connection pooling)
- SQL vs NoSQL selection criteria
- Testing database code (fixtures, transaction rollback, test databases)
- Common patterns (soft deletes, audit logs, pagination, hierarchical data)
